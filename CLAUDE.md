## Git Workflow

After completing any meaningful unit of work — a new feature, a bug fix, a styling change, a new section — commit and push to GitHub immediately. Do not batch unrelated changes into a single commit.

**Commit message format:**
- Use the imperative mood: "add hero section", "fix nav link styles", "update typography"
- Keep the subject line under 70 characters
- Be specific: describe *what* changed, not just "update files"

**After every commit, push:**
```
git push
```

This ensures work is never lost and the GitHub repo always reflects the latest state of the project.
